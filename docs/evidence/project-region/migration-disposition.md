# Migration-path disposition

The historical mixed-authority development trial and its runtime consumer chain have been deleted. Historical evidence remains under `docs/evidence/common-authority-full-trial` and the other migration evidence directories.

The only forward renderer is the whole-project canonical project-region path. It owns fresh placement, terminals, topology families, InterLayer slots, vertical/return columns, normalization, validation, coordinate ownership, bounds, and serialization. It never adapts legacy points and never accepts a partial project.

Normal production intentionally remains the legacy renderer. The explicit `--development-project-region` selector generates evidence and reports exact whole-project eligibility. It is disabled by default; an ineligible result falls back at the caller boundary, with no common/legacy route mixing.

Legacy corridor, traversal, candidate, and repair code may remain for normal production, but none is called by `GenerateProjectRegion`. The next safe deletion is therefore defect-driven retirement of legacy-only production responsibilities after a separately approved production selector, not another compatibility layer.
