Tranche 1 code-consolidation deletion record
============================================

Deleted immediately after compiler/reference audit:

- MovementScopeDefinition
  Projection-only record with no production, development, diagnostic, or test consumer.
  MovementScopeIdentity remains the canonical persisted movement identity.

- LinkConnectionFanoutGroup
  Empty aggregate model with no consumer. Fan-out behaviour consumes the ordered
  LinkConnectionFanoutMembership records directly.

Renamed rather than retained as compatibility aliases:

- RailDemand -> LinkSegmentDemand
- AssignedRail -> AssignedLinkSegment
- RailTransition -> LinkTransition
- DeterministicRailAllocator -> DeterministicSlotAllocator
- RouteInvalidation -> LinkInvalidation
- BandRouteDemand -> InterLayerLinkDemand
- BandConflictGrouper -> InterLayerConflictGrouper
- TerminalDemandCalculator -> LinkConnectionDemandCalculator
- InterLayerBandObserver -> InterLayerDemandDiscovery
- GroupedVerticalBandPlanner -> InterLayerSpacingConstraintProducer

No empty compatibility aliases were retained.

Audited and retained because they still have active consumers:

- Common-authority projection/parity models used by development diagnostics.
- DeterministicSharedTurnAllocator used by common-authority integration.
- DefectDemandContracts used by focused foundation tests and diagnostics.
- Legacy corridor, lane, traversal, repair, and candidate-selection types used by
  normal production through LegacyRoutingPipeline.

Audit method:

- compiler and focused test builds;
- repository-wide symbol occurrence counts across src and tests;
- explicit execution-entry searches for development trial and legacy production.
